# 奥読み深度採点表 v0.3

> 才覚診断のLLM出力（奥読み品質）を数値化する採点表。10項目×10点・減点方式・**95点以上で合格**（あつし基準）。
> v0.1（plans/okuyomi-depth-eval.md）に Codex (GPT-5.5) 独立レビュー20指摘のうち19を反映（2026-07-07）。保留1件は末尾「監視事項」。
> v0.3: GPT-5.6 Sol 独立レビュー（2026-07-12）で発見された採点ノイズ3種（skip付き減点の集計・judge自認の疑義減点・アンカー条件の片側適用）への対策 = 確定性ゲートを追加。
> 対象アプリ: **public** = saikaku-public-app（15問/105問の自由回答）/ **arch** = saikaku-architecture（TOP5単語リスト+深化3問）。

## 採点対象と原則

- 採点対象は**バリデータ通過後のユーザーが見る実物**（public: parseScoreResult 後 / arch: handler の JSON 抽出後）。
- **有効減点の4条件**（満たさない減点は無効・点数に反映しない。疑義はレポート注意欄へ）:
  1. 入力・出力からの**引用**が付いている（機械項目は機械的事実の提示で代替）
  2. サブアンカーの**所有者**（機械 or LLM）が明確で、越境減点でない
  3. **同一欠陥の二重減点でない**（重複排除規則は各項目に明記）
  4. **status: "confirmed" である**（確定性ゲート）— 採点者は各減点に confirmed / doubtful を必須で付け、アンカーの**全条件**（例: #1は「接続説明が出力内にない」ことまで）を確認できた場合のみ confirmed とする。doubtful・`skip:true`・note に「〜とまでは言えない/疑義/微妙/念のため/迷い」を含む減点はスクリプトが機械的に除外し notes へ退避する
- 混合項目はサブアンカー単位で所有者を固定。スクリプト減点とLLM減点を項目内で合算し、下限0にクリップ。**LLM採点者には機械採点結果を渡さず、LLM担当アンカーのみ提示**。最終合成は run-check 後にスクリプトが行う。

## 採点者の4契約

1. **まっさら起動**: 毎回 fresh context。改善履歴・前回スコアを渡さない（絶対評価）
2. **基準書換不可**: 本採点表は required schema で焼き込み。項目の増減 = 採点無効
3. **Makerと別の頭**: 生成 = claude-sonnet-4-6（本番同一設定）、採点 = claude-opus-4-8（temperature 0）
4. **実測強制**: 減点ごとに「断定文（出力引用）」と「不足している入力根拠」を引用。引用なき減点は無効

## 項目定義（10項目×10点）

### 1. 表面語からの奥読み〔LLM〕
- 軸名・kakuchiiki が、入力中の名詞句・動詞句を**同一語または同義語1語に置換しただけ**で、入力内の別回答との接続や「なぜそれを選ぶか」の説明が出力内にない: **-3/件**
- kakuchiiki **または kakuchiiki_options** に入力の固有対象がそのまま残る: **-4**（この欠陥は本項のみで減点。#5・#6では減点しない。v0.3.1: options も対象に — B5 の契約が options を含むため）
- 全軸が「入力語A→近い抽象語B」の置換のみで、**2つ以上の入力回答**を根拠にした動機・判断基準・反復パターンのいずれも述べていない: 本項上限3点

### 2. 根拠接地（捏造ゼロ）〔LLM+機械〕🔴致命
- 〔LLM〕入力から引用できる語句・出来事・選択傾向のいずれにも支えられない**具体的属性・経験・職業・対人傾向・過去の行動・感情状態の断定**: **-5/件**（2件で0点）。ただし出力内で「Aという根拠からBと推測する」と明示され、BがAの一般的解釈範囲に収まる場合は減点しない
- 〔機械 arch〕items の語が入力（TOP5+その他+深化回答）からの verbatim でない: **-3/件**
- 〔LLM public ja〕items の根拠フレーズが入力内容と意味対応せず、入力にない具体語を根拠として提示: **-3/件**
- 本項は「捏造」専用。捏造ではない一般論・テンプレは #9 で減点し、**同一文を両項で減点しない**

### 3. 横断統合〔LLM+機械〕
- 〔LLM public〕軸説明・items の根拠として引用できる入力箇所が**同一カテゴリ内の1回答しかなく**、他回答との接続が出力に示されていない軸: **-3/件**
- 〔LLM arch〕軸説明が TOP5 語のうち1語だけに依存し、他の TOP5 語または深化回答との接続が示されていない軸: **-3/件**
- 〔機械 arch〕TOP5 の語が items 全体に未収載（TOP5必須表示ルール違反）: **-2/語**

### 4. 掛け算演算〔LLM〕🔴致命
- kakuchiiki（と core_words の対応）が3要素を「Aで、Bで、Cな人」と**並列列挙するだけ**で、少なくとも2要素間の因果・制約・増幅関係が出力のどこにも1文以上で説明されていない: 本項上限4点
- 演算3問（各Noで **-3/問**）:
  1. 価値観が才能の使い方をどう制約・方向づけるかが読み取れるか
  2. 才能が情熱の追い方をどう具体化するかが読み取れるか
  3. 3要素が合わさることで単独要素からは出ない人物像・行動パターンが出ているか
- 3要素・複数軸間に明らかな**矛盾**があり、統合・条件分岐・緊張関係として説明されていない: **-4**（矛盾する2箇所以上を引用）
- kakuchiiki の人物像が、**最高 percentage の価値観・才能・情熱のうち2つ以上と対応していない**（低順位軸中心のキャッチコピー化）: **-5**（対応しない主軸名を引用）

### 5. 粒度の中庸・映像性〔LLM〕
- kakuchiiki・主要見出しの中心語が、具体的な行動・対象・場面を伴わない**抽象語のみ**: **-3/語**（対象語例: 世界・可能性・本質・未来・価値・成長・幸せ・自由・道・力・場）
- 「狭すぎる語」は入力固有対象**以外**の職業名・趣味名・場所名で人物像を狭めるもの: **-3/語**（入力固有対象の残留は #1 の担当）
- insight / description に、人物が「何を見て・誰に対して・何をするか」のうち**2要素以上が入力根拠に基づいて**書かれていない: **-4**
- 入力根拠にない場面・小道具で映像性を偽装: 本項で **-4**（#2では減点しない）

### 6. kakuchiiki 形式〔機械+LLM〕
- 〔機械 public〕16-32字でない: **-3** ／ 人物名詞で終わらない: **-4** ／ 読点2個以上: **-2** ／ options が2個未満・空文字・完全重複: **-2**
- 〔機械 arch〕20-35字でない: **-3** ／ 平易な人物名詞で終わらない: **-4** ／ options が3個未満・空文字・完全重複: **-2**
- 〔LLM〕options が本体と主要名詞・動詞の**7割以上を共有**し、意味差が形容詞差分だけ: **-2** ／ options の案が kakuchiiki 本体・上位軸のいずれとも意味的に対応しない: **-2/件**
- 人物名詞判定（機械）: 末尾が 者/人/家/師/士/役/手/主/匠/長/係/使い/職人/触媒/目利き/案内人/支援者/作り手/担い手/コーチ/ガイド/パートナー のいずれか（表記ゆれは辞書追記制。「目利き」は 2026-07-12 追加 — baseline 誤検知の解消）

### 7. 主軸の明確さ〔機械〕
- カテゴリ内 percentage の max-min ≤ 6pt: **-4/カテゴリ**
- カテゴリ内 percentage 合計が 100±15 逸脱: **-3/カテゴリ**

### 8. 軸名の規律〔機械 / LLM〕
- 〔機械 public ja〕価値観軸名が VALUE_LIST 完全一致でない: **-2/件** ／ 才能軸名が TALENT_LIST 完全一致でない: **-2/件**。ただし**表記ゆれ辞書**（本ファイル末尾、明示登録制）にある語は **-1/件**。辞書外の「近い語」判定は行わない
- 〔機械 arch〕軸名が2-6字でない: **-2/件**
- 〔LLM arch〕一般辞書にない語・独自連結の複合語・比喩のみで通常の人物特性として読めない語: **-3/件**（該当語を引用）

### 9. insight の深さとトーン〔LLM〕
- **根拠を伴わず人格を固定する断定文**（「あなたは必ず〜する人です」「〜に違いありません」）: **-3**。診断文体の自然な「あなたは〜です」は減点しない
- insight 段落のうち、入力からの具体語句・具体行動への参照が1つもなく、**別ペルソナにそのまま適用できる**一般助言・称賛・抽象説明のみの段落: **-3/段落**（捏造は #2 の担当、同一文の二重減点禁止）

### 10. 文の完結・字数遵守〔機械〕
- 〔public〕description が句点等で完結していない（途中切れ）: **-2/件**
- 〔共通〕必須フィールド欠落・空（axes/kakuchiiki/insight/summary/what/reward、arch は core_words も）: **-5**（v0.3.2: public にも what/reward を必須化 — バリデータは null 許容だがユーザー向け行動計画の欠落は品質欠陥）
- 〔共通〕フィールド字数が仕様上限超過: **-1/件**（最大-5）
- 〔arch〕kakuchiiki が文として読めない断片列: **-3**（このサブアンカーのみLLM所有）

## 総合とキャップ

- 総合 = 10項目の合計（各項目は0点で下限クリップ）
- **致命キャップ**: #2 または #4 の減点が**引用付き・confirmed の確定アンカー**によるもので項目点5点未満のとき、総合 = min(素点, 70)。文体・抽象度・テンプレ感による低得点ではキャップしない（#2=捏造断定、#4=掛け算不成立/kakuchiiki不整合に限定）
- 合格: **95点以上**。3周連続同点は打ち切り、採点軸を見直す
- **合否判定ラウンドの分散対策**（judge の run-to-run 分散 ±3-5点の実測に対応、2026-07-12）: fresh context で**3回独立採点**し、合否は3回の総合点**中央値**で決める。致命キャップは同一致命アンカー（#2/#4）が **2/3回以上再現**した場合のみ確定。3回の最高点−最低点とアンカー一致数をレポートに併記。通常ラウンドは1回でよい
- **production readiness の併記契約**: 合格はスコア95以上に加え「Sonnet 生成時間の p90 が baseline 比で悪化しない」「deep がタイムアウト予算内で完走する」を満たすこと（スコアのためにプロンプトを肥大させる Goodhart の歯止め）
